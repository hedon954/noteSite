# 5. Go 程序是如何启动起来的？

运行 runtime/rt0_xxx.s 汇编文件

  1. 读取命令行参数：复制参数变量 argc 和 argv 到栈上；
  2. 初始化 g0 执行栈：g0 是为了调度协程而产生的协程，是每个 Go 程序的第一个协程；
  3. 运行时检查：
    • 类型长度
    • 指针操作
    • 结构体字段偏移量
    • atomic 原子操作
    • CAS 操作
    • 栈大小是否是 2 的幂次
  4. 参数初始化 runtime.args：对命令行中的参数进行处理，参数数量复制给 argc（int32），参数值赋值给 argv（**byte）
  5. 初始化调度器 runtime.schedinit：
    • 全局栈空间内存分配
    • 堆内存空间的初始化
    • 初始化当前系统线程
    • 算法初始化（map、hash）
    • 加载命令行参数到 os.Args
    • 加载操作系统环境变量
    • gc 的参数初始化
    • 设置 process 的数量
  6. 创建主协程：执行 runtime.main 并将其放入调度器等待调度；
  7. 初始化一个 M，用来调度主协程；
  8. 主协程执行主函数 runtime.main，它会执行：
    • runtime.init
    • 启动 gc 垃圾回收器
    • 用户包依赖的 init 方法
    • 用户主函数 main.main
