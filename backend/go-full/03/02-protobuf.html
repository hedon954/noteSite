# ProtoBuf

ProtoBuf Github (opens new window)

ProtoBuf (opens new window)

ProtoBuf（Protocol Buffer）是 Google 的语言中立、平台中立、可扩展的结构化数据序列化机制。类似 XML，但是 ProtoBuf 更小、更快、更简单。一次性定义数据的结构方式，然后可以使用特殊生成的源代码轻松地使用各种语言在各种数据流中写入和读取结构化数据。

# 1. Download

  1. 在 Github 在寻找对应版本的 release 进行下载 protoc：https://github.com/protocolbuffers/protobuf/releases

  2. 解压

  3. 将 protoc 所在文件夹的 bin 目录配置进环境变量中，如 $HOME/protoc-3.19.1/bin

  4. 进入控制台，运行：

    protoc --version

    输出类似即表示已经安装完成：

    libprotoc 3.16.1

# 2. Go ProtoBuf

以下采用 Go 语言为演示代码。

# 2.1 定义 .proto 文件

syntax = "proto3";
package tutorial;

import "google/protobuf/timestamp.proto";
  • syntax 指定当前的 protobuf 语法版本。
  • package 是 ProtoBuf 的 package，不是 Go 的package，在后面的 gRPC 中用到。可以理解为定义一个包名，可以避免命名冲突。
  • import 表示引入其他的 proto 文件。
option go_package = "github.com/protocolbuffers/protobuf/examples/go/tutorialpb;tutorialpbForGo";
  • go_package 用来指定生成生成代码的路径即其包名：
    • github.com/protocolbuffers/protobuf/examples/go/tutorialpb：生成代码的路径
    • tutorialpbForGo：对应到 Go 中的 package
message Person {
  string name = 1;
  int32 id = 2;  // Unique ID number for this person.
  string email = 3;

  enum PhoneType {
    MOBILE = 0;
    HOME = 1;
    WORK = 2;
  }

  message PhoneNumber {
    string number = 1;
    PhoneType type = 2;
  }

  repeated PhoneNumber phones = 4;

  google.protobuf.Timestamp last_updated = 5;
}

// Our address book file is just one of these.
message AddressBook {
  repeated Person people = 1;
}
  • message 用来定义一个个的消息体，可以理解为一个类型。
  • enum 用来定义枚举类型。
  • repeated 用来定义复数，生成的 Go 代码会对应到 slice 类型。
  • =1、=2 的标志是表示当前字段是该 message 中的第 n 个字段，从 1 开始。

零值问题

ProtoBuf 中的所有字段都是可选的（方便新老系统的兼容）；

如果一个值没有设置，那么会默认设置为其的“零值”，故而在命名方面是需要考虑系统兼容性的，比如：

  • boolean fromNewUser（×）：老系统没有该字段，默认为 false，那么之前的用户数据就都是 Guest 了，不符合；
  • boolean fromGuest（√）：老系统没有该字段，默认为 false，之前的用户数据都判断为 false，也就是非 Guest，符合。
message HelloRequest {
	string ask = 1;
}

message HelloResponse {
	string answer = 1;
}

service HelloService {
    rpc SayHello (HelloRequest) returns (HelloResponse);
}

  • service 定义一个 RPC 服务

  • rpc 定义 RPC 服务中的一个接口，模板为：

    rpc MethodName (Request) returns (Response);

# 2.2 生成代码

安装代码生成器 protoc-gen-go：

go install google.golang.org/protobuf/cmd/protoc-gen-go@latest

运行下面命令，生成代码：

protoc -I=$SRC_DIR --go_out=$DST_DIR $SRC_DIR/addressbook.proto

  • -I: 指定 $SRC_DIR，也就是输入路径

  • --go_out: 指定 $DST_DIR，也就是输出路径，有一些可选参数：

    • paths=source_relative: 相对路径
    • plugins: 指定插件
    protoc -I=. --go_out=plugins=grpc,paths=source_relative:gen/go test.proto

  • 最后的空格后面指定 .proto 文件

# 2.3 使用代码

运行生成代码命令后，就可以在 .proto 中的 option go_package 中指定的路径中找到生成的代码，就可以使用 ProtoBuf API 了。
